# Koodo Reader 的 flomo 集成功能

本文档介绍了 flomo 集成功能，该功能允许用户将阅读高亮直接导出到他们的 flomo 账户。

## 概述

flomo 集成功能使 Koodo Reader 用户能够：
- 将选中的文本高亮导出到 flomo 账户
- 从图书库批量导出笔记和高亮
- 自动格式化高亮内容并包含图书元数据
- 配置和测试 flomo webhook 连接
- 无缝集成阅读笔记到 flomo 工作流程

## 功能特性

### 1. 高亮导出
- **一键导出**：选择文本并在弹出菜单中点击"导出到 flomo"
- **丰富格式**：自动格式化高亮内容，包含书名、作者、章节和位置信息
- **智能内容**：包含阅读上下文和标签，便于更好的组织

### 2. 图书库批量导出
- **右键菜单**：从图书库右键菜单访问批量导出选项
- **导出笔记**：批量导出指定书籍的所有笔记
- **导出高亮**：批量导出指定书籍的所有高亮
- **全部导出**：一次操作导出笔记和高亮
- **进度反馈**：批量操作期间的实时进度更新
- **智能分批**：自动限速防止API过载

### 3. 配置管理
- **Webhook URL 设置**：在设置中轻松配置 flomo webhook URL
- **连接测试**：内置测试功能验证 webhook 连接性
- **格式验证**：自动验证 webhook URL 格式

### 4. 错误处理
- **网络错误处理**：优雅处理网络连接问题
- **用户反馈**：清晰的成功/失败消息，提供可操作的信息
- **配置验证**：防止使用无效配置进行导出尝试

## 设置说明

### 前提条件
- flomo PRO 账户（webhook 功能必需）
- 活跃的互联网连接

### 配置步骤

1. **获取您的 flomo webhook URL**：
   - 访问 [flomo webhook 设置](https://flomoapp.com/mine?source=incoming_webhook)
   - 复制您的专属 webhook URL（应以 `https://flomoapp.com/mine/` 开头）

2. **配置 Koodo Reader**：
   - 打开 Koodo Reader 设置
   - 导航到"flomo"选项卡
   - 启用 flomo 集成
   - 粘贴您的 webhook URL
   - 点击"保存"
   - 使用"测试连接"验证设置

3. **开始导出高亮**：
   - 在阅读时选择任意文本
   - 在弹出菜单中点击"导出到 flomo"
   - 检查您的 flomo 账户中的导出笔记

## 使用方法

### 导出单个高亮

1. **选择文本**：在书中高亮任意文本
2. **访问导出菜单**：弹出菜单将显示各种选项
3. **点击"导出到 flomo"**：如果已配置，flomo 导出选项将可用
4. **确认**：您将收到成功/失败通知

### 从图书库批量导出

1. **进入图书库**：前往您的图书库/收藏
2. **右键点击书籍**：右键点击任意书籍打开上下文菜单
3. **选择导出选项**：从以下选项中选择：
   - **导出笔记到 flomo**：导出该书的所有笔记
   - **导出高亮到 flomo**：导出该书的所有高亮
   - **全部导出到 flomo**：同时导出笔记和高亮
4. **监控进度**：观看导出过程中的实时进度更新
5. **完成通知**：接收成功/失败导出的汇总信息

#### 批量导出功能特性
- **智能分批**：自动在导出间添加延迟以防止API限速
- **进度跟踪**：导出过程的实时反馈
- **错误处理**：单个项目失败不会停止整个过程
- **统计显示**：导出前显示笔记/高亮数量

### 内容格式

导出的高亮遵循以下格式：

```
📖 《书名》- 作者姓名

"您的高亮文本内容"

📍 章节名称 | 页码：42 | 位置：25%
🏷️ #读书笔记 #KoodoReader
```

### 设置管理

- **启用/禁用**：开启或关闭 flomo 集成
- **Webhook URL**：配置您的专属 flomo webhook URL
- **测试连接**：验证您的 webhook 是否正常工作
- **清除配置**：如需要可移除 webhook URL

## 技术实现

### 架构

flomo 集成由几个组件组成：

1. **FlomoService** (`src/utils/service/flomoService.ts`)：
   - 处理 API 通信的核心服务
   - 内容格式化和验证
   - 错误处理和用户反馈

2. **PopupOption 组件** (`src/components/popups/popupOption/component.tsx`)：
   - 与文本选择弹出菜单集成
   - 用户交互处理
   - 导出触发功能

3. **FlomoSetting 组件** (`src/containers/settings/flomoSetting/`)：
   - 配置界面
   - Webhook URL 管理
   - 连接测试

### API 集成

集成使用 flomo 的 webhook API：
- **方法**：POST
- **Content-Type**：application/json
- **载荷**：`{ "content": "格式化笔记", "source": "Koodo Reader" }`

### 配置存储

设置使用 Koodo Reader 的 ConfigService 存储：
- `isEnableFlomo`：功能启用的布尔标志
- `flomoWebhookUrl`：用户的 webhook URL

## 错误处理

集成包含全面的错误处理：

### 网络错误
- 连接超时
- DNS 解析失败
- 服务器不可用

### 配置错误
- 无效的 webhook URL 格式
- 缺少 webhook URL
- 集成被禁用

### 用户反馈
- 导出期间的加载状态
- 成功确认
- 详细的错误消息
- 可操作的错误指导

## 安全考虑

- **Webhook URL**：作为敏感数据处理，本地存储
- **仅 HTTPS**：只接受 HTTPS webhook URL
- **无数据记录**：导出内容不被记录或存储
- **本地配置**：所有设置存储在用户设备本地

## 故障排除

### 常见问题

1. **"flomo 未配置"**：
   - 确保在设置中启用了 flomo 集成
   - 验证 webhook URL 已正确配置

2. **"无效的 webhook URL 格式"**：
   - 检查 URL 是否以 `https://flomoapp.com/mine/` 开头
   - 确保从 flomo 设置中正确复制了 URL

3. **"连接测试失败"**：
   - 验证互联网连接
   - 检查 flomo 服务是否可访问
   - 确认 webhook URL 仍然有效

4. **"发生网络错误"**：
   - 检查互联网连接
   - 稍后重试
   - 验证 flomo 服务状态

### 调试步骤

1. 在浏览器中直接测试 webhook URL
2. 验证 flomo PRO 账户状态
3. 检查 Koodo Reader 控制台的详细错误消息
4. 尝试禁用并重新启用集成

## 限制

- 需要 flomo PRO 账户
- 导出需要互联网连接
- 仅限文本内容（无图片或格式）
- Webhook URL 必须手动配置
