# 导出功能更新说明

## 更新概述

基于用户反馈和代码维护考虑，我们对导出功能进行了重大更新，移除了存在中文支持问题的PDF导出功能，并优化了现有的导出选项。

## 主要变更

### ❌ 移除的功能

1. **PDF导出功能**
   - 移除了 `PDFExportService` 类
   - 移除了所有PDF相关的导出选项
   - 移除了 `jspdf` 依赖包
   - 原因：jsPDF 3.0.1版本对中文Unicode字符支持极其有限，导致严重的乱码问题

### ✅ 保留和优化的功能

1. **文本格式导出**
   - 完美支持中文字符
   - 保持结构化格式
   - 包含所有元数据信息
   - 文件命名：`${书名}-笔记导出-${数量}条.txt`

2. **Anki格式导出**
   - 支持生成Anki卡片格式
   - 统一文件命名规则
   - 文件命名：`${书名}-笔记导出-${数量}条.txt`

3. **CSV格式导出**
   - 适合数据分析和处理
   - 兼容Excel等表格软件

4. **Flomo格式导出**
   - 支持批量导出到Flomo

## 技术改进

### 1. 代码简化
- 移除了复杂的PDF生成逻辑
- 减少了代码维护成本
- 提高了系统稳定性

### 2. 依赖优化
- 移除了 `jspdf` 依赖（减少包大小约100KB）
- 保留了必要的 `file-saver` 依赖

### 3. 文件命名统一
- 所有导出格式使用一致的命名规则
- 格式：`${书名}-笔记导出-${数量}条.{扩展名}`
- 完美支持中文文件名

## 用户影响

### 对现有用户的影响

1. **PDF导出用户**：
   - 不再能直接导出PDF格式
   - 建议使用文本导出，然后通过其他工具转换为PDF
   - 推荐工具：Microsoft Word、Google Docs、Markdown编辑器等

2. **中文用户**：
   - 获得更好的导出体验
   - 所有导出格式完美支持中文
   - 不再有乱码问题

### 替代方案

如果用户需要PDF格式，可以：

1. **使用文本导出 + 转换**：
   ```
   导出文本 → 复制到Word/Google Docs → 导出为PDF
   ```

2. **使用Markdown编辑器**：
   ```
   导出文本 → 转换为Markdown → 使用Typora等工具导出PDF
   ```

3. **使用在线转换工具**：
   ```
   导出文本 → 使用在线文本转PDF工具
   ```

## 使用指南

### 文本导出使用方法

1. 点击书籍操作菜单中的"导出"
2. 选择"Text"格式
3. 选择要导出的内容类型（笔记/高亮/全部）
4. 点击导出

### 导出内容格式

```
书名
作者: 作者名
导出时间: 2024-01-01 12:00:00
笔记数量: 10条

==================================================

1. "这是高亮文本"
笔记: 这是我的笔记内容
章节: 第一章
位置: 25%

------------------------------

2. "另一段高亮文本"
笔记: 另一段笔记内容
章节: 第二章
位置: 50%

------------------------------
```
